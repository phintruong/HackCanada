--------------------------------------------------------------------------------
1. CIVILIAN MODE (Patient Routing)
--------------------------------------------------------------------------------

INCORPORATED:
  [x] Location detection — browser geolocation + postal code input
  [x] Postal code geocoding via Mapbox (reverse geocode for "Use my location")
  [x] Vitals capture — heart rate, respiratory rate, stress index (VitalsCapture)
  [x] Symptom intake — structured symptom cards (chest pain, shortness of breath,
      fever, dizziness, etc.) and free text (SymptomCards)
  [x] AI-powered triage — GPT-4o-mini classifies severity (critical / urgent / non-urgent)
  [x] Multi-factor smart routing — scores hospitals by:
      - Live driving time (Mapbox Directions API, driving-traffic profile)
      - Time-of-day / rush hour adjustments (temporalPatterns)
      - ER wait time adjustments (hourly patterns)
      - Hospital occupancy (congestion data)
      - Specialty matching (cardiac, trauma, stroke, etc.)
      - Severity-based weight profiles (critical vs urgent vs non-urgent)
  [x] Top 3 recommendations with drive time, wait time, total time, reasoning
  [x] Route visualization — recommended route (green), alternatives (dashed gray) on map
  [x] User location marker — pulsing blue dot on map
  [x] 911 button — shown for critical severity (RoutingResult, tel:911)
  [x] Traffic timeline — congestion segments along route, time prediction, reroute suggestions

NOT INCORPORATED / PARTIAL:
  [ ] None identified for core civilian flow; all README-listed features are present.

--------------------------------------------------------------------------------
3. MAP VISUALIZATION
--------------------------------------------------------------------------------

INCORPORATED:
  [x] Mapbox GL — single map at /map (no separate map-3d route; map-3d removed per plan)
  [x] Dark theme and 3D view (pitch, bearing)
  [x] Hospital congestion circles — color-coded by occupancy (CongestionLayer)
  [x] Live congestion data — Ontario Open Data API with MongoDB mock fallback (congestionService)
  [x] Hospital 3D footprints — OSM-based footprints (HospitalFootprintsLayer);
      data: public/map-data/hospital-footprints-toronto.geojson
  [x] Iconic landmarks 3D — fill-extrusion with height overrides (LandmarksLayer);
      data: landmarks-toronto.geojson, landmarks-mississauga.geojson, landmarks-kitchener.geojson
  [x] No global “all buildings” layer — addGlobalBuildings: false in createMap
  [x] Map cleanup on unmount — map.remove() to avoid WebGL leaks
  [x] Shared map init — lib/mapbox/createMap.ts with getFirstSymbolLayerId for layer order

NOT INCORPORATED / PARTIAL:
  [ ] Hospital footprint GeoJSON only exists for Toronto; other cities (e.g. Mississauga, Kitchener,
      Waterloo) do not have hospital-footprints-{city}.geojson — run scripts/fetchHospitalFootprints.ts
      per city if 3D hospital buildings are desired there.
  [ ] Realistic 3D landmark models (e.g. CN Tower as detailed GLB) — plan notes current landmarks
      are simplified extruded polygons; true-to-life models would require custom Mapbox layer (e.g.
      Three.js/deck.gl) and are noted as a possible later enhancement.

SUMMARY
--------------------------------------------------------------------------------

  Most README features are incorporated:
  - Civilian: location, vitals, symptoms, triage, routing, top 3, routes on map, 911, timeline.
  - Government: place ER, capacity, Voronoi, before/after, flow arcs, blueprints, suitable parcels.
  - Map: Mapbox, dark, 3D pitch, hospital + landmark extrusions, congestion circles, no global buildings.
  - APIs: hospitals, congestion, triage, route, simulate, seed, editor building, speak, street-sound.
  - Editor: /editor with GLB export and building APIs.

  Gaps or follow-ups:
  1. Generate hospital-footprints-{city}.geojson for non-Toronto cities if 3D hospitals are needed.
  2. Realistic 3D landmark models (e.g. CN Tower) would need custom layer work (future).
  3. Traffic/spawner lib is present; confirm whether full vehicle simulation is required on /map.
  4. Editor: clarify if tree editing is part of scope and document accordingly.
